### OCP
- 개방 폐쇄 원칙
- 코드의 수정 없이 기능 수정하거나 추가할 수 있어야 함

예전에는 OCP하면 Getter와 Setter를 떠올랐다.  
하지만 요즘은 Decorator Pattern과 DI가 떠오른다.
----
Decorator 패턴은 말 그대로 장식자 역할을 한다.  
메서드를 감싸고 그 메서드가 실행하기 전 또는 실행 한 후에 일정 기능을 수행할 수 있도록 해준다.

하지만 이건 메서드를 일급 객체로 취급하는 언어에서만 구현이 가능하다.

----
DI는 DIP가 이루어진 상황에서 제일 빛을 발한다고 생각한다.

인터페이스를 구현한 객체에서 메서드의 방식을 좀 만 바꾸어서 DI를 하면 코드의 수정없이 기능을 수정 한 것 아닌가?
